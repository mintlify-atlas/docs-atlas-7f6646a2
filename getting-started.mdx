Welcome to the Hack Western Platform! This guide will walk you through setting up your local development environment and getting the platform running on your machine.

## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Operating System**: Linux, MacOS, or Windows with [Windows Subsystem For Linux](https://learn.microsoft.com/en-us/windows/wsl/about)
- **Node.js**: Version 21 or higher with npm (we recommend using [nvm](https://github.com/nvm-sh/nvm))
- **Docker**: [Docker Desktop](https://www.docker.com/products/docker-desktop/) or an alternative such as [colima](https://github.com/abiosoft/colima)
- **Git**: Installed and configured with a GitHub account
- **Modern Browser**: Chrome 108+, Edge 108+, Firefox 108+, Safari 16+, or similar
- **Text Editor**: VSCode or Neovim with TypeScript LSP, Prettier, and ESLint plugins configured to run on save/commit

<Note>
You should have already been added to the Hack Western GitHub organization to access the repository.
</Note>

## Quick start

<Steps>

<Step title="Clone the repository">

Clone the Hack Western repository and navigate into it:

```bash
git clone https://github.com/your-org/hackwestern.git
cd hackwestern
```

</Step>

<Step title="Install dependencies">

Install all project dependencies using npm:

```bash
npm install
```

This will install all packages defined in `package.json`, including Next.js, React, tRPC, Drizzle ORM, and more.

</Step>

<Step title="Configure environment variables">

Get a copy of the `.env` file from a web lead, or create your own by copying the example file:

```bash
cp .env.example .env
```

At minimum, you'll need to generate a `NEXTAUTH_SECRET`:

```bash
openssl rand -base64 32
```

Copy the output and add it to your `.env` file:

```bash
NEXTAUTH_SECRET="your-generated-secret-here"
NEXTAUTH_URL="http://localhost:3000"
DATABASE_URL="postgresql://postgres:password@localhost:5432/."
```

<Warning>
Never commit your `.env` file to version control. It contains sensitive secrets and is already included in `.gitignore`.
</Warning>

</Step>

<Step title="Start the database">

Start the development PostgreSQL database using Docker:

```bash
npm run db:start
```

This runs the `start-database.sh` script, which launches a PostgreSQL container configured for local development.

</Step>

<Step title="Run database migrations">

Push the Drizzle database schema to your dev database:

```bash
npm run db:migrate
```

This applies all database migrations to create the necessary tables and structure.

</Step>

<Step title="Seed the database (optional)">

Populate the database with fake data for testing:

```bash
npm run db:seed
```

<Tip>
Seeding is optional but recommended for development. It creates sample data you can use to test features without manual data entry.
</Tip>

</Step>

<Step title="Start the development server">

Launch the Next.js development server:

```bash
npm run dev
```

The platform will be available at [http://localhost:3000](http://localhost:3000).

</Step>

</Steps>

## Managing the database

The Hack Western Platform uses [Drizzle ORM](https://orm.drizzle.team) with PostgreSQL. Here are the key database commands:

### View and edit data

Open Drizzle Studio to manage your database with a visual interface:

```bash
npm run db:studio
```

### Making schema changes

When you need to modify the database schema:

<Steps>

<Step title="Generate migration files">

After modifying schema files, generate SQL migration scripts:

```bash
npm run db:generate
```

</Step>

<Step title="Apply migrations">

Apply the generated migrations to your database:

```bash
npm run db:migrate
```

</Step>

</Steps>

<Warning>
Do NOT use `npx drizzle-kit push` anymore. Always generate and apply migrations using the commands above to maintain a proper migration history.
</Warning>

## Additional configuration

### Authentication providers

The platform supports multiple OAuth providers. To enable them, add the following to your `.env` file:

```bash
# GitHub OAuth
GITHUB_CLIENT_ID="your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"

# Google OAuth
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"

# Discord OAuth
DISCORD_CLIENT_ID="your-discord-client-id"
DISCORD_CLIENT_SECRET="your-discord-client-secret"
```

### Email service (Resend)

To enable email functionality, add your Resend API key:

```bash
RESEND_API_KEY="your-resend-api-key"
```

### File storage (Cloudflare R2)

For resume uploads and other file storage features, configure Cloudflare R2:

```bash
R2_ACCESS_KEY_ID="your-access-key"
R2_SECRET_ACCESS_KEY="your-secret-key"
R2_ENDPOINT="https://<accountid>.r2.cloudflarestorage.com"
R2_BUCKET_NAME="hw12-resumes"
R2_PUBLIC_BASE_URL="https://pub-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.r2.dev"
```

## Troubleshooting

### Database connection issues

If you can't connect to the database, ensure Docker is running and the container is started:

```bash
docker ps
```

You should see a PostgreSQL container running. If not, restart it with `npm run db:start`.

### Port already in use

If port 3000 is already in use, you can specify a different port:

```bash
PORT=3001 npm run dev
```

### Node version issues

Verify you're using Node 21 or higher:

```bash
node --version
```

If you're using nvm, switch to the correct version:

```bash
nvm install 21
nvm use 21
```
